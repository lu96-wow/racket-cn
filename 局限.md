# 局限 — racket-cn 无法翻译的模块

## 展开器时序限制（2 个）

这些模块导出的 form 在 rename 绑定建立**之前**就被展开器消费，无法改名。

| 模块 | 导出 | 消费阶段 |
|------|------|---------|
| `racket/load` | `#%module-begin`, `#%top-interaction` | Module-begin |
| `racket/for-clause` | `syntax-local-splicing-for-clause-introduce` | kernel 展开 |

### 原理

```
源码 → Reader → Module-begin（消费 require, #%module-begin, define-syntax-rule）
                ↓
           Binding 解析（rename-out 此时才生效）
                ↓
           局部展开（所有绑定就绪）
```

任何在 Binding 解析之前被消费的 form 都无法通过 rename-out 或 make-rename-transformer 改名。
与 `require` 不能用 `引用` 替代是同一根本原因。

## 模块层限制（3 个 form）

| form | 中文 | 替代方案 |
|------|------|---------|
| `require` | `引用` | 模块顶层必须用 `require`；但其子 form 全部可用 |
| `define-syntax-rule` | `定义语法规则` | 用 `定义语法` + `语法匹配` |
| `#%module-begin` | — | 无法替代 |

## 纯内部/编译器模块（9 个）

这些模块没有用户可直接使用的独立绑定，全部是展开器/编译器内部实现：

| 模块 | 说明 |
|------|------|
| `racket/init` | 启动引导，重导出整个 racket/base |
| `racket/kernel` | kernel 原语（+, -, * 等），已由 base-impl 覆盖 |
| `racket/interactive` | REPL 内部，无 phase-0 导出 |
| `racket/phase+space` | 展开器 phase/空间内部 |
| `racket/provide-transform` | 展开器 provide-transform 内部 |
| `racket/require-transform` | 展开器 require-transform 内部 |
| `racket/stxparam-exptime` | stxparam 展开期内部（已由 stxparam 覆盖 phase-1） |
| `racket/unit-exptime` | unit 展开期内部（已由 unit 覆盖） |
| `racket/for-clause` | for 循环展开器的内核钩子（#%kernel 层） |

> 注：`linklet.rkt`、`provide.rkt`、`require.rkt`、`provide-syntax.rkt`、`require-syntax.rkt` 已翻译为中文子模块（提供 `define-provide-syntax`、`matching-identifiers-out` 等子 form 的中文别名），不再是纯内部模块。
